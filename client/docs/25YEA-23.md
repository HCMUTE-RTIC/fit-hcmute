# Task 25YEA-23: [FE_Admin] CMS Gallery - Quản lý Album & Batch Upload

## 1. Bối cảnh & Mục tiêu

Xây dựng phân hệ quản lý Thư viện ảnh (Gallery) trong trang Admin CMS. Cho phép biên tập viên (EDITOR) tổ chức hình ảnh theo các Album (Bộ ảnh) với cấu hình SEO tối ưu và công cụ upload hàng loạt (Batch Upload) tiện lợi.

## 2. Yêu cầu chi tiết

- **Trang danh sách Album**:
  - Hiển thị lưới/bảng danh sách các Album.
  - Hỗ trợ tìm kiếm, lọc và các hành động hàng loạt.
- **Trang biên tập Album (Thêm mới/Chỉnh sửa)**:
  - Nhập thông tin cơ bản: Tiêu đề, Mô tả.
  - **Cấu hình SEO**: Tự động sinh Slug, nhập Meta Title, Meta Description (Theo quy chuẩn `/docs/news_seo_sequence.md`).
  - **Upload Media**: Giao diện Drag & Drop cho phép ném hàng chục ảnh vào cùng lúc.
  - **Preview**: Hiển thị ảnh đang upload và kết quả sau khi upload thành công.
- **Kỹ thuật**:
  - Sử dụng `gallery.service.ts` để giao tiếp với Backend NestJS.
  - Batch upload trực tiếp lên MinIO thông qua API xử lý luồng (Stream).
  - Phân quyền: EDITOR và SUPER_ADMIN.

### [2026-02-28] Bước 4: Chức năng Tải ảnh hàng loạt (Drag & Drop Batch Upload)

- **File thay đổi**: Tách riêng UI phức tạp vào file `components/admin/BatchUpload.tsx`, cập nhật `gallery.service.ts`.
- **Mô tả những công việc đã làm**:
  1. **Tạo Component Tách biệt**: Nhằm tái sử dụng cho các trang khác nếu cần, `BatchUpload` xử lý độc lập việc bắt sự kiện kéo thả thẻ HTML5 (`onDragOver`, `onDrop`).
  2. **Xử lý Tiến độ Giao diện (Progress Tracking)**: Cứ mỗi file được kéo vào, component sinh ra một `UploadTask` mang cờ trạng thái, hiển thị một thanh Progress Bar riêng biệt trực quan. Liên kết với hàm `uploadBatchMedia` trong Service nhận callback `(progress, index)` để tăng dần phần trăm.
  3. **Bypass Lỗi Type Strict**: Cấu trúc mảng file `for...of` ở phía Service được viết lại cẩn thận thay cho vòng lặp `for-i` truyền thống để Next.js compiler hiểu luồng dữ liệu an toàn, không báo lỗi "Object is possibly 'undefined'".

### [2026-02-28] Bước 5: Cải thiện Trải nghiệm Lập trình & Điều hướng Hệ thống

- **File thay đổi**: `services/gallery.service.ts`, `components/admin/Sidebar.tsx`.
- **Mô tả những công việc đã làm**:
  1. **Khắc phục lỗi Next.js Runtime Overlay (Bug Fix)**: Trong môi trường dev, hàm `fetch` khi gọi API mock sẽ sinh ra lỗi `Failed to fetch`. Lập trình viên trước đó dùng `console.error(error)` trong block `catch` để in ra. Vì cơ chế mới của Next.js rất nhạy cảm, nó văng hẳn màn hình đỏ "Unhandled Runtime Error". **Giải pháp**: Đã thay bằng `console.warn` để vẫn giữ thông báo trên console nhưng không ép trình duyệt hiện màn hình đỏ.
  2. **Mở rộng Sidebar Menu**: Truy cập file `Sidebar.tsx`, tại biến định tuyến nội bộ, Thư mục "Thư viện Ảnh" đã được chuyển từ một liên kết đơn (Link) thành một Cây phân cấp (Dropdown) bổ sung mục "Danh sách" và "Thêm Album mới". Giúp định hướng User rõ ràng.
  3. **Tối ưu hóa Active State**: Refactor lại logic tự động bung Dropdown trong móc `useEffect`, chỉ cần dùng hàm `startsWith()` là đáp ứng đủ cho đường dẫn bất kỳ mà không cần code các ngoại lệ rườm rà. Lệnh `pnpm run typecheck` xác nhận dự án hiện tại hoàn toàn sạch lỗi đánh máy lập trình (Type-Safe).
